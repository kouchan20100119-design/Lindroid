# Lindroid - デザイン仕様書

## アプリコンセプト

**Lindroid**は、Android端末上でLinux環境を実行し、開発者がターミナル操作とGUIアプリケーションの両方を利用できるモバイルアプリです。iOS Human Interface Guidelines (HIG)に準拠し、モバイルファーストの縦向き(9:16)、片手操作を前提とした設計を行います。

## ターゲットユーザー

- モバイル環境で開発作業を行いたい開発者
- サーバー管理やスクリプト実行をモバイルで行いたいエンジニア
- Linux環境を学習・実験したい技術者

## 主要機能

1. **ターミナルエミュレータ**: 基本的なLinuxコマンドの実行
2. **GUI対応**: VNC/X11プロトコルを通じたGUIアプリケーションの表示
3. **セッション管理**: 複数のターミナルセッションの保存と切り替え
4. **ファイルマネージャー**: Linux環境内のファイル操作

## 画面構成

### 1. Home Screen (ホーム画面)
**目的**: アプリの起動ポイントとして、主要機能へのアクセスを提供

**コンテンツ**:
- アプリタイトルとロゴ
- 「New Terminal Session」ボタン(新規ターミナルセッション開始)
- 「Saved Sessions」リスト(保存済みセッションのカード表示)
- 「GUI Viewer」ボタン(VNC/X11ビューアへの遷移)
- 「File Manager」ボタン(ファイルマネージャーへの遷移)

**レイアウト**:
- 上部: アプリロゴとタイトル
- 中央: 大きな「New Terminal Session」ボタン
- 下部: 保存済みセッションのスクロール可能なカードリスト

### 2. Terminal Screen (ターミナル画面)
**目的**: Linuxコマンドの実行とターミナル操作

**コンテンツ**:
- ターミナル出力エリア(黒背景、緑または白文字)
- コマンド入力フィールド(画面下部固定)
- ツールバー(画面上部):
  - 戻るボタン
  - セッション名表示
  - 保存ボタン
  - 設定ボタン(フォントサイズ、カラースキーム)

**レイアウト**:
- 全画面表示のターミナルエミュレータ
- 下部にソフトキーボード連携の入力フィールド
- スワイプジェスチャーでツールバーの表示/非表示

### 3. GUI Viewer Screen (GUIビューア画面)
**目的**: VNC/X11プロトコルを通じてLinux GUIアプリケーションを表示

**コンテンツ**:
- VNC/X11接続設定フォーム(初回のみ):
  - ホストアドレス
  - ポート番号
  - 認証情報(オプション)
- GUI表示エリア(全画面)
- タッチジェスチャーでマウス操作をエミュレート
- ツールバー:
  - 戻るボタン
  - 接続/切断ボタン
  - 仮想キーボード表示ボタン

**レイアウト**:
- 全画面でGUI表示
- ピンチズームとパンジェスチャー対応
- 2本指タップで右クリックエミュレート

### 4. File Manager Screen (ファイルマネージャー画面)
**目的**: Linux環境内のファイルシステムの閲覧と操作

**コンテンツ**:
- 現在のディレクトリパス表示
- ファイル/フォルダのリスト表示(アイコン、名前、サイズ、更新日時)
- ツールバー:
  - 戻るボタン
  - 新規フォルダ作成ボタン
  - ファイルアップロードボタン
- 各アイテムのコンテキストメニュー(長押し):
  - 開く
  - 削除
  - リネーム
  - 共有

**レイアウト**:
- 上部: パンくずリストでディレクトリ階層表示
- 中央: スクロール可能なファイルリスト
- 下部: ツールバー

### 5. Settings Screen (設定画面)
**目的**: アプリ全体の設定とカスタマイズ

**コンテンツ**:
- ターミナル設定:
  - フォントサイズ
  - カラースキーム(ダーク/ライト/カスタム)
  - キーボードショートカット
- GUI設定:
  - デフォルト接続先
  - 画質設定
- 一般設定:
  - テーマ(ライト/ダーク/システム)
  - 言語設定

**レイアウト**:
- セクション分けされた設定項目リスト
- 各項目はタップで詳細設定画面へ遷移

## ユーザーフロー

### メインフロー1: 新規ターミナルセッション開始
1. ホーム画面で「New Terminal Session」ボタンをタップ
2. ターミナル画面に遷移
3. コマンドを入力・実行
4. 必要に応じてセッションを保存(保存ボタンタップ)
5. 戻るボタンでホーム画面に戻る

### メインフロー2: GUI表示
1. ホーム画面で「GUI Viewer」ボタンをタップ
2. GUIビューア画面に遷移
3. 初回の場合、接続設定を入力
4. 接続ボタンをタップしてVNC/X11サーバーに接続
5. GUI画面が表示され、タッチ操作でマウスをエミュレート
6. 戻るボタンで接続を切断してホーム画面に戻る

### メインフロー3: ファイル操作
1. ホーム画面で「File Manager」ボタンをタップ
2. ファイルマネージャー画面に遷移
3. フォルダをタップして移動、ファイルをタップして開く
4. 長押しでコンテキストメニューを表示し、削除やリネームを実行
5. 戻るボタンでホーム画面に戻る

## カラースキーム

### ブランド色
- **Primary**: `#00D084` (鮮やかなグリーン - Linuxのイメージカラー)
- **Background (Light)**: `#FFFFFF`
- **Background (Dark)**: `#0D1117` (GitHub Darkに近い深い黒)
- **Surface (Light)**: `#F6F8FA`
- **Surface (Dark)**: `#161B22`
- **Foreground (Light)**: `#24292F`
- **Foreground (Dark)**: `#E6EDF3`
- **Muted (Light)**: `#57606A`
- **Muted (Dark)**: `#8B949E`
- **Border (Light)**: `#D0D7DE`
- **Border (Dark)**: `#30363D`

### ターミナル専用色
- **Terminal Background**: `#0D1117` (ダーク固定)
- **Terminal Foreground**: `#58A6FF` (明るいブルー)
- **Terminal Cursor**: `#00D084` (プライマリカラー)

## タイポグラフィ

- **ヘッダー**: SF Pro Display Bold, 24-32pt
- **ボディテキスト**: SF Pro Text Regular, 16-18pt
- **ターミナルテキスト**: SF Mono Regular, 14pt (等幅フォント)
- **キャプション**: SF Pro Text Regular, 12-14pt

## インタラクションデザイン

### タップフィードバック
- ボタン: スケール(0.97) + ライトハプティック
- リストアイテム: 不透明度(0.7)
- アイコン: 不透明度(0.6)

### ジェスチャー
- **ターミナル画面**: 上スワイプでツールバー表示/非表示
- **GUIビューア画面**: ピンチズーム、パン、2本指タップで右クリック
- **ファイルマネージャー**: 左スワイプで削除、長押しでコンテキストメニュー

## 技術的制約

- React Native + Expo環境での実装
- ターミナルエミュレータはWebViewまたはネイティブモジュールで実装
- VNC/X11クライアントはReact Nativeライブラリまたはカスタムネイティブモジュールを使用
- Linux環境はTermux APIまたは類似のソリューションを活用(実際のLinuxカーネルは不要、エミュレーション層で対応)

## 実装優先順位

1. **Phase 1**: ホーム画面とターミナル画面の基本UI
2. **Phase 2**: ターミナルエミュレータの機能実装(コマンド実行、出力表示)
3. **Phase 3**: セッション管理機能(保存、読み込み)
4. **Phase 4**: GUIビューア画面とVNC/X11クライアント統合
5. **Phase 5**: ファイルマネージャー画面
6. **Phase 6**: 設定画面とカスタマイズ機能
